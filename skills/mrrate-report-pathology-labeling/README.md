## What This Skill Does

This skill helps Codex prepare and inspect the MR-RATE pathology classification
source scripts. It explains the source pathology JSON, CoT-to-JSON
cross-validation, PRESENT-label verification, per-rank JSON outputs, and merged
label CSV.

## Why You Would Call It

Call this skill when:

- You have structured MR-RATE report CSVs and need pathology labels.
- You need to inspect the source-defined 37-label set.
- You need to merge `labels_rank_*.json` into `labels.csv`.

Use it to:

- Prepare classification and merge commands.
- Explain label provenance and output metadata.
- Keep research-label caveats visible.

Do not use it when:

- Structure QC has not passed and the user has not accepted the risk.
- The task is diagnosis, triage, treatment, or clinical decision-making.

## How It Works

The classification script loads `batch00_reports.csv` through
`batch27_reports.csv` when present, reads `study_uid` and `findings`, classifies
each report against the source pathology JSON, cross-validates JSON with CoT
reasoning, verifies PRESENT labels, and writes per-rank JSON. The merge script
converts those per-rank JSON files to a `study_uid` by pathology CSV.

## API And Options

Source commands:

```text
srun python 06_pathology_classification/classify_pathologies_parallel.py --reports_dir /path/to/reports --pathologies_json 06_pathology_classification/data/pathologies.json --output_dir /path/to/output --model_name Qwen/Qwen3.5-35B-A3B --batch_size 500 --seed 42
```

```text
python 06_pathology_classification/merge_labels.py --input_dir /path/to/output --output /path/to/labels.csv
```

Configuration:

- Default seed in source: `42`.
- Default model in source: `Qwen/Qwen3.5-35B-A3B`.
- Pathology definitions: `06_pathology_classification/data/pathologies.json`.
- Optional mapping provenance: `pathologies_snomed_map.json`.

## Inputs And Outputs

Inputs can include:

Outputs can include:

- `labels_rank_{rank}.json`.

Output locations:

- User-supplied `--output_dir`.
- User-supplied merge `--output`.

## Limitations

Known limitations:

- Labels are generated by an LLM workflow and manual validation context from the source README; they are not clinical ground truth.
- The skill does not add or modify the source label ontology.
- No sample report fixtures are packaged with this skill.

Choose another skill when:

- Reports still need structuring or structure QC.
- You only need generic CSV/JSON merge help.

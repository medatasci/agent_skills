## Work Performed

- Continued the adapter-skeleton implementation lane for
  `codebase-to-agentic-skills`.
- Added `--from-scan-json` support to the skill-specific
  `scaffold-adapter` helper so a review-only adapter skeleton can be generated
  from a specific scanner output candidate instead of only from a generic
  adapter-policy type.
- Added candidate selection by `--candidate-id` or `--candidate-index`.
- Added adapter-plan stub selection by `--stub-type` or `--stub-index`.
- Preserved the selected stub's source refs, required reviews, guardrails,
  confirmation requirement, suggested commands, and smoke-test notes inside the
  generated review-only adapter script.
- Made `--adapter-name` optional when using `--from-scan-json`; the helper now
  derives a deterministic script name from the candidate and adapter type when
  no explicit name is provided.
- Updated the top-level SkillForge CLI command
  `python -m skillforge codebase-scaffold-adapter` to pass the new scan-backed
  options through to the skill helper.
- Added tests for both the generic scaffold mode and the new scan-backed mode,
  including the top-level CLI path.
- Updated requirements, user docs, Python module docs, skill docs, and the
  whitepaper to describe source-grounded adapter scaffolding.
- Rebuilt generated catalog, plugin, static site, and search-index artifacts.

  copied plugin artifacts are regenerated; future review tooling could make
  this easier to inspect.

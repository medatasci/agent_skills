## What Went Well

- The new extractor is source-grounded: every command has a path, line, snippet, and side-effect risk instead of being guessed by the LLM.
- The candidate-level CLI draft now has a stronger evidence chain from source file to command hint to adapter planning.
- The tests exercise both documented command snippets and Python source entrypoint detection, which should catch future regressions in healthcare repo scanning.

## What Could Be Improved

- The extractor intentionally uses conservative heuristics. It does not yet parse full `argparse`, Click, or Typer option schemas.
- Platform detection is useful but shallow; future work could distinguish Windows PowerShell examples, POSIX shell examples, Docker commands, WSL assumptions, and Python virtual-environment setup more precisely.
- Side-effect risk is keyword-based. Healthcare skills would benefit from a richer side-effect classifier that separates installs, downloads, writes, GPU/model runs, PHI/data access, and network calls.

## Next Action

- Add side-effect-category grouping and risk summaries to `command_evidence` so candidate skill reviewers can quickly separate read-only inspection commands from installs, downloads, file writes, GPU/model execution, and network access.

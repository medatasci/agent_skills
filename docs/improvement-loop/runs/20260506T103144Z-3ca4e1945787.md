## What Went Well

- The focused fixture now tests the exact user experience concern: a multi-workflow repo should not make every candidate show the same command order.
- The generated report became easier to review because the command evidence column includes actual command text.
- The candidate table now communicates why commands were ignored for a candidate recommendation, which should help humans understand the scanner's choices.

## What Could Be Improved

- The relevance scorer is still term based. It does not understand synonyms beyond the predefined task terms, and it can miss commands whose relationship is only visible through surrounding documentation.
- Some candidates fall back to workflow evidence when no candidate-specific command terms are found. That is conservative, but future work could add source-section proximity or heading-aware scoring.
- The generated wide candidate table remains dense; candidate-specific detail pages or collapsible sections would make it easier to review larger repos.

## Next Action

- Add heading-aware and source-section-aware command relevance so commands inherit context from the README/doc section they appear under, not only from the command text itself.

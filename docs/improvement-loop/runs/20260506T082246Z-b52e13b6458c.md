## Work Performed

- Continued the real-repo hardening work for `codebase-to-agentic-skills`.
- Investigated why the previous NV-Generate-CTMR real scan reported
  `source_version.status: not-a-git-repo` even though the checkout contains a
  `.git` directory.
- Found the cause: Git rejected the checkout with a `dubious ownership`
  safe-directory error because the sandbox user differs from the directory
  owner.
- Added a safe, read-only retry path to `git_commit()`: when Git provenance
  lookup fails because of safe-directory or dubious-ownership checks, the
  scanner retries only that command with `git -c safe.directory=<repo> ...`.
  It does not write global Git config or modify the source checkout.
- Added `safe_directory_override` and a distinct
  `ok-safe-directory-override` status to `source_version` so reviewers know how
  the provenance was obtained.
- Improved provisional candidate naming by scoping candidate names with the
  source project name when available. For the real NV-Generate scan, the first
  candidate is now `NV-Generate-CTMR synthetic medical image workflow` instead
  of only `Research synthetic medical image workflow`.
- Preserved the generic task name as `generic_candidate_skill` for traceability.
- Re-ran the real NV-Generate scan and regenerated the scan-backed review-only
  guarded-run adapter scaffold with the source-scoped candidate id.
- Updated requirements, docs, and skill instructions to describe source-scoped
  candidate names and safe-directory provenance behavior.
- Rebuilt generated catalog, plugin, static site, and search artifacts.
